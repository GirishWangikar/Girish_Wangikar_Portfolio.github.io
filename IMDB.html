IMDB Review Binary Classifier

Dataset and Preprocessing The first step of the project involves downloading the IMDB Reviews dataset using TensorFlow Datasets. The dataset will be loaded as a collection of (input, label) pairs, where the input represents the movie review text and the label represents whether the review is positive or negative.

Next, the text data will be preprocessed. The sentences will be tokenized and padded to a uniform length using the Tokenizer and pad_sequences functions from TensorFlow's Keras. The preprocessing ensures that all reviews have the same length, which is essential for training the neural network.

Building the Model
 The sentiment classification model will be constructed using TensorFlow's Keras API. The model architecture includes an Embedding layer, which represents each word in the vocabulary with trainable word vectors. These vectors will be learned as the model trains. The output of the Embedding layer will be flattened and then passed through a Dense layer with a ReLU activation function. Finally, a single neuron with a sigmoid activation function will be used as the output layer to classify the reviews as positive or negative.

Training the Model
 The model will be trained using the binary cross-entropy loss function and the Adam optimizer. The training process will be executed for a specified number of epochs. After training, the model's performance will be evaluated on the test set to measure its accuracy in predicting sentiment.

Visualizing Word Embeddings
 After training, the word embeddings learned by the Embedding layer will be visualized using the TensorFlow Embedding Projector. The word embeddings will be saved to files vecs.tsv and meta.tsv, which will contain the vector weights of each word in the vocabulary and the corresponding words, respectively. These files will be loaded into the Tensorflow Embedding Projector to explore how words are clustered based on their learned representations.
